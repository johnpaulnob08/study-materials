Networking Today

Explain the advances in modern technologies. Covers network components, topologies, connection types, reliable network principles, trends, and security fundamentals.

Lessons

All Lessons

1.1 Networks Affect Our Lives Explain how networks affect our daily lives ▼
Key Ideas
  • Communication is almost as important as air, water, food, and shelter.
  • Through the use of networks, we are connected like never before.
  • Networks create a world without boundaries — global communities and a human network.
1.2 Network Components Explain how host and network devices are used ▼
Host Roles
  • Every computer on a network is called a host or end device.
  • Servers — provide information to end devices (email, web, file).
  • Clients — send requests to servers to retrieve information.
  • Peer-to-Peer — a device acts as both client and server. Only recommended for very small networks.
Peer-to-Peer Comparison
AdvantagesDisadvantages
Easy to set upNo centralized admin
Less complexNot as secure
Lower costNot scalable
Simple tasks (file transfer, printing)Slower performance
End Devices
Where a message originates from or is received. Data originates with an end device, flows through the network, and arrives at an end device.
Intermediary Devices
  • Interconnects end devices (switches, APs, routers, firewalls)
  • Regenerate and retransmit data signals
  • Maintain network pathway information
  • Notify devices of errors
Network Media Types
TypeSignal
Metal wire (copper)Electrical impulses
Fiber-optic cablePulses of light
WirelessElectromagnetic waves
1.3 Network Representations and Topologies Explain network representations and topology diagrams ▼
Network Representations
  • Network diagrams (topology diagrams) use symbols to represent devices.
  • NIC — Network Interface Card
  • Physical Port — connector/outlet on a device
  • Interface — specialized port on a network device
  • Note: "port" and "interface" are often used interchangeably.
Topology Diagrams
  • Physical Topology — illustrates the physical location of intermediary devices and cable installation.
  • Logical Topology — illustrates devices, ports, and the addressing scheme of the network.
1.4 Common Types of Networks Compare characteristics of common network types ▼
Networks by Size
  • Small Home Networks — connects a few computers + internet
  • SOHO (Small Office/Home Office) — connects home/remote office to corporate network
  • Medium to Large Networks — hundreds or thousands of interconnected computers across many locations
  • World Wide Networks — hundreds of millions of computers (e.g., the Internet)
LAN vs WAN
LANWAN
Small geographical areaWide geographical area
Single org / individualOne or more service providers
High-speed bandwidthSlower speed links
Interconnects end devicesInterconnects LANs
The Internet
  • Worldwide collection of interconnected LANs and WANs.
  • Not owned by any individual or group.
  • Governed by: IETF, ICANN, IAB
Intranet vs Extranet
  • Intranet — private LANs/WANs internal to an organization, accessible only to members.
  • Extranet — provides secure access to outside individuals who need access to organizational data.
1.5 Internet Connections Explain how LANs and WANs interconnect to the internet ▼
Home & Small Office Connections
TypeDescription
CableHigh bandwidth, always on, via cable TV provider

Converged Networks
  • Before: telephone, video, and data networks were separately cabled using different technologies and standards.
  • Converged networks carry data, voice, and video over the same infrastructure using the same set of rules and standards.
1.6 Reliable Networks Describe the four basic requirements of a reliable network ▼
Fault Tolerance
  • Limits the impact of a failure by limiting the number of affected devices.
  • Multiple paths are required for fault tolerance.
  • Packet switching — splits traffic into packets routed over a network; each can take a different path.
  • Contrast: Circuit-switched networks establish dedicated circuits (no redundancy).
Scalability
  • A scalable network can expand quickly and easily to support new users and applications.
  • Must not impact performance of services to existing users.
  • Network designers follow accepted standards and protocols to enable scalability.
Quality of Service (QoS)
  • Voice and live video require higher delivery expectations.
  • QoS is the primary mechanism to ensure reliable delivery for all users.
  • With a QoS policy, the router manages the flow of data and voice traffic by priority.
Network Security
  • Infrastructure Security — physical security of devices + preventing unauthorized access.
  • Information Security — protecting data transmitted over the network.
  • Three Goals: Confidentiality, Integrity, Availability (CIA)
1.7 Network Trends Explain BYOD, collaboration, video, and cloud computing trends ▼
BYOD (Bring Your Own Device)
  • Allows users to use personal devices — laptops, tablets, smartphones, e-readers.
  • Gives users more freedom and flexibility to access information anywhere.
  • BYOD = any device, any ownership, used anywhere.
Online Collaboration
  • Collaborate and work with others over the network on joint projects.
  • Tools like Cisco WebEx / Webex Teams allow: instant messaging, images, videos, links.
  • A very high priority for businesses and education.
Cloud Computing
  • Store files or access applications via servers over the internet.
  • Made possible by data centers. Smaller companies lease cloud services.
  • Public Cloud — pay-per-use or free, available to general public
  • Private Cloud — for a specific organization (e.g., government)
  • Hybrid Cloud — mix of two or more cloud types
  • Custom Cloud — built for a specific industry (healthcare, media)
Other Trends
  • Smart Home Technology — appliances interconnect (e.g., oven + calendar).
  • Powerline Networking — uses electrical wiring to send data; useful where Wi-Fi can't reach.
  • Wireless Broadband (WISP) — ISP connects subscribers via access points; uses cellular tech; useful in rural areas.
1.8 Network Security Identify security threats and solutions for all networks ▼
Threat Types
  • External: Viruses, worms, Trojan horses
  • External: Spyware and adware
  • External: Zero-day attacks, Denial of Service attacks
  • External: Data interception, identity theft
  • Internal: Lost/stolen devices
  • Internal: Accidental misuse by employees
  • Internal: Malicious employees
Security Solutions
  • Security must use multiple layers — more than one solution.
  • Home/SOHO: Antivirus, antispyware, firewall filtering
  • Larger Networks: Dedicated firewall systems
  • Larger Networks: Access Control Lists (ACL)
  • Larger Networks: Intrusion Prevention Systems (IPS)
  • Larger Networks: Virtual Private Networks (VPN)
1.9 The IT Professional Explain employment opportunities in networking ▼
CCNA Certification
  • Demonstrates knowledge of foundational technologies.
  • Ensures relevance for adoption of next-gen technologies.
  • New CCNA focus: IP foundation, security, wireless, virtualization, automation, network programmability.
  • DevNet certifications available at associate, specialist, and professional levels.
Networking Jobs
  • Find employment opportunities at netacad.com → Careers → Employment Opportunities.
  • Use the Talent Bridge Matching Engine to find jobs with Cisco, Cisco partners, and distributors.

Basic Switch and End Device Configuration

Implement initial settings including passwords, IP addressing, and default gateway parameters on a network switch and end devices using Cisco IOS CLI.

Lessons

All Lessons

2.1 Cisco IOS Access Explain how to access a Cisco IOS device ▼
OS Components
  • Shell — user interface (CLI or GUI)
  • Kernel — communicates between hardware & software; manages hardware resources
  • Hardware — the physical part of the computer
GUI vs CLI
  • GUI — graphical icons, menus, windows (Windows, macOS, iOS, Android). More user-friendly.
  • GUIs can fail or crash — for this reason, network devices are typically accessed via CLI.
  • CLI requires keyboard input; output viewed on monitor.
Access Methods
  • Console — physical management port; used for initial configs.
  • SSH — secure remote CLI connection. Recommended method.
  • Telnet — insecure remote CLI; passwords sent in plaintext. Avoid.
  • Terminal Emulators: PuTTY, Tera Term, SecureCRT
2.2 IOS Navigation Explain how to navigate Cisco IOS modes ▼
Primary Command Modes
  • User EXEC Mode (>) — limited monitoring commands only.
  • Privileged EXEC Mode (#) — access to all commands and features.
Configuration Submodes
  • Global Config Mode — access configuration options on the device.
  • Line Config Mode — configure console, SSH, Telnet, or AUX access.
  • Interface Config Mode — configure a switch port or router interface.
Navigation Commands
  1. Switch> enable # Enter Privileged EXEC
  2. Switch# configure terminal # Enter Global Config
  3. Switch(config)# line console 0 # Enter Line Config
  4. Switch(config-line)# exit # Return to Global Config
  5. Switch(config)# end # Return to Privileged EXEC
  6. Switch(config)# Ctrl+Z # Return to Privileged EXEC
2.3 The Command Structure Describe the Cisco IOS command structure ▼
Command Syntax
  • Keyword — specific parameter defined in the OS (e.g., ip protocols)
  • Argument — user-defined value (e.g., 192.168.10.5)
ConventionMeaning
boldfaceEnter literally as shown
italicsSupply your own value
[x]Optional element
{x}Required element
[x {y | z}]Required choice within optional element
IOS Help & Shortcuts
  • ? — context-sensitive help (available commands, arguments)
  • Command shortening — e.g., conf t for configure terminal
  • Tab — completes a partial command
  • Up Arrow / Ctrl+P — recalls previous commands
  • Ctrl+C / Ctrl+Z — exits configuration mode
  • Ctrl+Shift+6 — break sequence (abort ping/traceroute/DNS)
2.4 Basic Device Configuration Configure a Cisco IOS device using CLI ▼
Device Naming Rules
  • Start with a letter
  • No spaces
  • End with a letter or digit
  • Only letters, digits, and dashes
  • Less than 64 characters
  1. Switch(config)# hostname S1
Password Guidelines
  • More than 8 characters in length
  • Mix of upper/lowercase, numbers, special characters
  • Avoid using the same password for all devices
  • Do not use common words
Configuration Commands
  1. # Secure User EXEC (console port)
  2. S1(config)# line console 0
  3. S1(config-line)# password cisco
  4. S1(config-line)# login
  5. # Secure Privileged EXEC
  6. S1(config)# enable secret class
  7. # Secure VTY lines (Telnet/SSH remote access)
  8. S1(config)# line vty 0 15
  9. S1(config-line)# password cisco
  10. S1(config-line)# login
  11. # Encrypt all plaintext passwords
  12. S1(config)# service password-encryption
  13. # Banner message (# is the delimiter)
  14. S1(config)# banner motd #Authorized Access Only!#
2.5 Save Configurations Use IOS commands to save the running configuration ▼
Configuration Files
  • startup-config — stored in NVRAM. Loaded on startup/reboot. Persistent (survives power off).
  • running-config — stored in RAM. Current active config. Lost when device powers off.
Save / Restore Commands
  1. # Save running config to startup
  2. S1# copy running-config startup-config
  3. # Reload the device
  4. S1# reload
  5. # Erase startup config
  6. S1# erase startup-config
  7. # View running config
  8. S1# show running-config
2.6 Ports and Addresses Explain how devices communicate across network media ▼
IPv4 Addressing
  • IPv4 address — 32-bit, dotted decimal notation (4 groups of 0–255, e.g., 192.168.1.1)
  • Subnet mask — 32-bit value differentiating network portion from host portion.
  • Default gateway — IP address of the router used to reach remote networks.
IPv6 Addressing
  • 128-bit address; written as 32 hexadecimal values.
  • Groups of 4 hex digits separated by colons (:)
  • Not case-sensitive (uppercase or lowercase).
  • IPv6 is the most recent version, replacing IPv4.
Network Media Considerations
  • Types: twisted-pair copper, fiber-optic, coaxial, wireless
  • Differences: distance the signal can travel, environment, data speed requirements, cost of media and installation
2.7 Configure IP Addressing Configure a host device with an IP address ▼
Manual vs Automatic IP
  • Manual — statically enter IPv4 address, subnet mask, and default gateway in Windows: Control Panel → Network → Adapter → Properties → TCP/IPv4
  • Automatic (DHCP) — device automatically receives IP config from DHCP server. Default behavior for most end devices.
  • IPv6: uses DHCPv6 or SLAAC (Stateless Address Autoconfiguration).
Switch Virtual Interface (SVI)
To access the switch remotely, configure an IP on the SVI (VLAN 1 by default):
  1. S1(config)# interface vlan 1
  2. S1(config-if)# ip address 192.168.1.20 255.255.255.0
  3. S1(config-if)# no shutdown
2.8 Verify Connectivity Verify connectivity between two end devices ▼
Verification Commands
  1. # Test end-to-end connectivity
  2. S1# ping 192.168.1.10
  3. # View brief interface status
  4. S1# show ip interface brief
  5. # Check IP config on Windows PC
  6. C:\> ipconfig
Interface Assignment Steps
  • Connect console cable from PC to switch.
  • Open terminal emulation program, accept defaults to reach CLI.
  • Use enable to enter Privileged EXEC mode.
  • Use interface config mode and enter no shutdown to activate an interface.
  • Use ping to test end-to-end connectivity between switches and PCs.

Protocols and Models

Explain how network protocols enable devices to access local and remote network resources — covering communication rules, protocol suites, standards organizations, OSI/TCP-IP models, encapsulation, and data access.

Lessons

All Lessons

3.1 The Rules Describe the types of rules necessary to communicate ▼
Communication Fundamentals
  • It is not enough to have a connection — devices must agree on how to communicate.
  • Three elements: Source (sender), Destination (receiver), Channel (media)
  • Protocols must account for: identified sender/receiver, common language, speed & timing, confirmation/acknowledgment.
Network Protocol Requirements
  • Message encoding — converting info into transmittable form; decoding reverses it.
  • Formatting & encapsulation — specific format depending on message type and channel.
  • Message size — converted to bits; encoded as light, sound, or electrical impulses.
  • Message timing — flow control, response timeout, access method.
  • Delivery options — unicast, multicast, broadcast.
Unicast
One-to-one. One sender, one specific receiver.
Multicast
One-to-many (not all). Sender sends to a specific group of devices.
Broadcast
One-to-all. Used in IPv4 networks only. IPv6 uses Anycast instead.
3.2 Protocols Explain why protocols are necessary in network communication ▼

3.3 Protocol Suites Explain the purpose of adhering to a protocol suite ▼
What is a Protocol Suite?
  • A group of inter-related protocols necessary to perform a communication function.
  • Sets of rules that work together to solve a problem.
  • Viewed in terms of layers — higher layers and lower layers (lower layers move data, upper layers provide services).
Protocol Suites History
  • TCP/IP — most common; maintained by IETF. Open standard, freely available.
  • OSI Protocols — developed by ISO and ITU.
  • AppleTalk — proprietary, released by Apple Inc.
  • Novell NetWare — proprietary, developed by Novell Inc.
TCP/IP Suite Characteristics
  • Used by the internet; includes HTTP, TCP, IP, SMTP, FTP, DNS, DHCP, ARP, etc.
  • Open standard — freely available, used by any vendor.
  • Standards-based — endorsed by the networking industry and approved by standards organizations.
  • Protocols operate at the application, transport, and internet layers.
  • Network access layer: Ethernet (802.3) and WLAN (802.11).
3.4 Standards Organizations Explain the role of standards organizations in network protocols ▼
Internet Standards Bodies
  • ISOC — Promotes open development and evolution of the internet
  • IAB — Management and development of internet standards
  • IETF — Develops, updates, and maintains internet and TCP/IP technologies
  • IRTF — Long-term internet and TCP/IP research
  • ICANN — Coordinates IP address allocation, domain name management
  • IANA — Oversees IP addresses, domain names, and protocol identifiers for ICANN
Electronic & Communications Standards
  • IEEE — Standards in power, healthcare, telecom, and networking (802.3, 802.11)
  • EIA — Electrical wiring, connectors, 19-inch rack standards
  • TIA — Radio, cellular towers, VoIP, satellite communication standards
  • ITU-T — Video compression, IPTV, broadband (DSL) standards
3.5 Reference Models Explain how TCP/IP and OSI models facilitate standardization ▼
OSI Model — 7 Layers
#LayerDescription
7ApplicationProcess-to-process communications
6PresentationCommon data representation
5SessionManages data exchange
4TransportSegment, transfer, reassemble data
3NetworkExchange data over the network (IP)
2Data LinkExchange frames over common media (MAC)
1PhysicalActivate/maintain physical connections
TCP/IP Model — 4 Layers
LayerDescription
ApplicationRepresents data to user; encoding & dialog control
TransportCommunication between devices across diverse networks
InternetDetermines the best path through the network
Network AccessControls hardware devices and media
Key Comparison
OSI Layers 1&2 = TCP/IP Network Access. OSI Layers 5,6,7 = TCP/IP Application. OSI has 7 layers; TCP/IP has 4.
Why Use a Layered Model?
  • Assists protocol design — defined interfaces between layers
  • Fosters competition — products from different vendors can interoperate
  • Technology changes in one layer do not affect other layers
  • Provides a common language to describe networking functions and capabilities
3.6 Data Encapsulation Explain how data encapsulation transports data across the network ▼
Segmentation & Sequencing
  • Segmentation — breaking messages into smaller units.
  • Multiplexing — interleaving multiple segmented data streams.
  • Benefits: increases speed (no link monopolization) and increases efficiency (only failed segments retransmitted).
  • Sequencing — numbering segments so message can be reassembled. TCP is responsible for sequencing.
Protocol Data Units (PDUs)
  • Encapsulation is a top-down process — each layer adds its header.
  • Data — Application layer stream
  • Segment — Transport layer (TCP/UDP header added)
  • Packet — Network layer (IP header added)
  • Frame — Data Link layer (MAC header + trailer added)
  • Bits — Physical layer (transmitted as signals)
De-encapsulation (receiving end)
  • Data is de-encapsulated as it moves up the stack at the receiver.
  • Each layer strips off its header and passes data to the layer above.
  • Received order: Bits → Frame → Packet → Segment → Data
3.7 Data Access Explain how local hosts access local and remote network resources ▼
Layer 3 — IP Addressing (Logical)
  • Source IP — sending device; original source.
  • Destination IP — receiving device; final destination.
  • IP address = Network portion (left; identifies the network) + Host portion (right; identifies the device).
  • L3 IP addressing stays the same across all hops (source to destination).
Layer 2 — MAC Addressing (Local)
  • Delivers frames on the same network segment (link).
  • MAC addresses are physically embedded in the Ethernet NIC.
  • Destination MAC is always on the same link as source, even if ultimate destination is remote.
  • L2 MAC addressing changes at every hop (link-to-link).
Same Network
  • Source and destination share the same network portion of the IP address.
  • Frame uses the actual destination MAC address directly.
  • Example: PC1 (192.168.1.110) → FTP Server (192.168.1.9)
Remote Network
  • Source and destination have different network portions.
  • Default Gateway (DGW) — the router interface IP on the local LAN; the "door" to remote networks.
  • Layer 2 frame is addressed to the router's MAC address at each hop.
  • L3 IP stays the same; L2 MAC changes at every hop.
