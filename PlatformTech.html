Topic 01

Platform Technology Introduction

What platforms are, how computing evolved, and what makes up IT infrastructure.
Key Definition

A Platform is any combination of Hardware + Software + Networks that forms a base on which other technologies or services are built. Think of it as the foundation — almost everything you use (websites, apps, games) runs on top of a platform.

What Makes Up a Platform?
  • Hardware Environment — The physical computer, server, or device
  • Software — Programs designed to run within that hardware
  • Associated Facilities — Networks, tools, and resources used alongside it
Real-World Example: Your smartphone is a platform. Apps (Instagram, TikTok) are built on top of it. The phone's hardware + Android/iOS = the platform.
5 Generations of Computing
1945–1955 · Gen 1

Vacuum Tubes — Huge machines using glass tubes. Programmed with punch cards and machine language. Very slow and unreliable.

1955–1965 · Gen 2

Transistors & Batch Systems — Transistors replaced tubes. Jobs were collected in batches and run together. Introduced mainframes like IBM 1401, IBM 7094 and FORTRAN compilers.

1965–1980 · Gen 3

Integrated Circuits (ICs) & Multiprogramming — Multiple programs could run in memory at once. Introduced IBM System/360, OS/360, timesharing, and MULTICS.

1980–Present · Gen 4

Personal Computers — IBM PC, DOS, Macintosh, Windows, Linux. Computing came to homes and offices.

1990–Present · Gen 5

Mobile Computers — PDAs, Symbian OS, BlackBerry, Smartphones, iOS, Android. Computing in our pockets.

What is an Operating System (OS)?
Definition

An Operating System is a collection of software that manages computer hardware and provides services for programs. It's like a manager — it controls the CPU, memory, disk, and all devices.

  • The most important program on a computer — loaded first when you turn it on
  • Acts as an intermediary between users and hardware
  • Every app (browser, game, Word) needs an OS to run
  • Manages resources: CPU time, memory space, disk access
Two Main Roles of an OS
Extended Machine
Hides ugly hardware details and gives programmers clean, easy-to-use tools and abstractions.
Resource Manager
Shares and manages CPU, memory, and I/O devices among multiple programs running at the same time.
Types of Operating Systems
Mainframe OS Server OS Multiprocessor OS Personal Computer OS Handheld OS Embedded OS Real-time OS Smart Card OS
IT Infrastructure — 7 Main Components

Everything a company needs to run its technology systems:

1
Computer Hardware Platforms Client machines (PCs, laptops), servers, blade servers, mainframes. Key makers: Intel, AMD, IBM, HP, Dell.
2
Operating System Platforms Servers mostly run Unix/Linux (65%) or Windows (35%). Clients mostly Windows. Mobile: Android, iOS.
3
Enterprise Software Applications Large-scale business software. Providers: SAP, Oracle. Middleware: BEA.
4
Data Management & Storage Database software (IBM DB2, Oracle, MySQL, SQL Server). Physical storage (EMC, Seagate, Western Digital). SANs connect multiple storage devices.
5
Networking Platforms Network OS (Windows Server, Linux). Hardware providers: Cisco, Alcatel-Lucent. Telecoms: AT&T, Verizon.
6
Internet Platforms Hardware/software for websites, intranets, extranets. Server market: IBM, Dell, HP. Web tools: Microsoft .NET, Oracle Java, Adobe.
7
Consulting & System Integration Services Ensures new and old systems work together. Key firms: Accenture, IBM Global Services, Infosys, Wipro.
Evolution of IT Infrastructure (Eras)
1959–Present

Mainframe & Minicomputer Era — IBM mainframes (1958), DEC minicomputers (1965)

1981–Present

Personal Computer Era — IBM PC introduced; personal software boom

1983–Present

Client/Server Era — Desktop clients networked to servers; work split between them

1992–Present

Enterprise Computing Era — Integration of networks using Internet standards

2000–Present

Cloud & Mobile Era — Computing delivered over the Internet; smartphones dominate

  • A platform = Hardware + Software + Networks that others build on
  • Computing evolved through 5 generations, from vacuum tubes to smartphones
  • An OS manages hardware resources and acts as a bridge between users and hardware
  • IT Infrastructure has 7 components: hardware, OS, enterprise software, data storage, networking, internet, and consulting
  • IT evolved from mainframes → PCs → client/server → enterprise → cloud/mobile

Platform Technology: Windows

Windows history, GUI, file system structure, NTFS permissions, and command-line tools.

1981

MS-DOS — Command-line only. Users typed commands to interact with the computer.

1985

Windows 1.0 — First graphical user interface (GUI). Click and interact visually.

1990/1992

Windows 3.0/3.1 — Introduced virtual memory and games like Solitaire.

1995

Windows 95 — Big milestone: introduced the Start button. Looked like modern Windows.

1998

Windows 98 — Improved USB support.

2001

Windows XP ⭐ — One of the best and most stable versions. Widely used for many years.

2006

Windows Vista ❌ — Disliked for being buggy and prone to crashes.

2009

Windows 7 ⭐ — Restored trust after Vista. Major migration target from XP.

2012/2013

Windows 8 / 8.1 — Built for tablets; removed (then restored) the Start bar.

2015

Windows 10 — Intended to be the final version; continuous updates.

2021

Windows 11 — Current stable version; regularly updated for security.

Windows GUI — Key Elements
  • Taskbar — Bottom bar with pinned apps, open programs, system icons, search bar, widgets (weather), and task tray (background apps)
  • Desktop — Main workspace for opening apps and storing files/shortcuts. Windows 11 adds virtual desktops for separate workspaces
  • Start Menu — Accessed via Start button; shows all applications and a powerful search
  • Personalization — Right-click desktop → "Personalize" to change backgrounds, colors, and themes
Windows OS Folder Structure

Understanding what's inside your C: drive:

  • File Explorer — Primary tool for navigating files and folders
  • C: Drive — Where Windows is installed and all data is stored
  • Users Folder — Contains folders for each user (Documents, Pictures, Desktop). "Public" folder accessible to all users
  • Program Files — Stores 64-bit applications (faster, more efficient)
  • Program Files (x86) — Stores older 32-bit applications
  • Windows Folder — Core OS files. Do not delete!
  • System32 Folder — Critical for Windows to run. Deleting it can completely break the system
  • SysWOW64 — Like System32 but for 64-bit systems
  • WinSxS Directory — Windows component store; contains updates and service packs
  • AppData (Hidden) — Stores app settings per user; useful for troubleshooting
Windows File Systems — How Windows Organizes Data
What is a File System?

A file system is the method an OS uses to organize and store data on a storage device. Every storage device (phone, USB, hard drive) uses one.

File System Year Max File Size Key Notes
FAT
File Allocation Table		 1977 4 GB (FAT32) Very old, limited capacity. FAT32 caps disk at 32 GB. Legacy
HPFS
High Performance File System		 ~1988 — Briefly used after FAT. Not used today. Obsolete
NTFS
New Technology File System		 1993 16 Exabytes Required for Windows installs. Supports file recovery & permissions. Windows-only. Standard
exFAT
Extended File Allocation Table		 2006 16 Exabytes Cross-platform (works on Windows, Mac, Linux). Ideal for USB drives & SD cards. Portable
NTFS Permissions — Controlling Who Can Access Files

To view: right-click any file/folder → Properties → Security tab

PermissionWhat it Allows
Full ControlAll possible actions on the file or folder
Read & ExecuteOpen and run files, but cannot change them
Read OnlyView the file only — cannot edit or run
ModifyRead, write, and run — but cannot change other users' permissions
WriteAdd or change data in a file
DenyBlocks all access — overrides all other permissions
⚠️ Important: Files inside a folder inherit the folder's permissions by default. But you can set explicit permissions on individual files to override this. Deny always takes precedence!

📌 Topic 2 Quick Summary
  • Windows evolved from MS-DOS (1981) text commands → Windows 11 (2021) modern GUI
  • Best versions: Windows XP (stability), Windows 7 (recovery), Windows 10/11 (current)
  • NTFS is the standard file system for Windows — supports huge files and permissions
  • exFAT is best for removable storage (USB, SD cards) — works across all OS platforms
  • NTFS permissions control file access: Full Control, Read, Write, Modify, Deny
  • "Deny" always overrides all other permissions — most powerful restriction

Topic 03

Virtualization & Containerization

Running multiple systems on one machine, hypervisors, containers, Docker, and Kubernetes.
Key Definition

Virtualization is a technology that lets you create multiple virtual machines (VMs) on a single physical computer. Instead of buying 5 physical servers, you run 5 virtual servers on one machine. It separates hardware from software environments.

Benefits of Virtualization
  • Cost Savings — Fewer physical machines needed = less money spent on hardware
  • Better Utilization — Use 100% of a server's power instead of leaving it idle
  • Easier Backup & Recovery — Save and restore VM states quickly
  • Faster Provisioning — Spin up a new system in minutes, not hours
  • Improved Isolation — One VM crashing doesn't affect others
What is a Hypervisor?
Definition

A Hypervisor is software that creates, runs, and manages virtual machines. It acts as a layer between the physical hardware and the virtual systems running on it.

Type 1 Hypervisor

Runs directly on the hardware — no host OS needed.

  • High performance
  • Used in data centers
  • No host OS required
VMware ESXi Microsoft Hyper-V Xen
Type 2 Hypervisor

Runs on top of an existing OS (like Windows).

  • Easy to install
  • Great for training & testing
  • Lower performance vs Type 1
Oracle VirtualBox VMware Workstation
Virtual Machine Lifecycle — How to Create a VM
1
Create VM on Hypervisor — Start a new virtual machine in your hypervisor software
2
Assign Resources — Allocate CPU cores, RAM amount, and storage size to the VM
3
Install Guest OS — Install Windows, Linux, or any OS inside the VM
4
Configure Network — Set up networking so the VM can communicate
5
Start the VM — Power it on and begin using it like a real computer
VM Management Tools: Snapshots (save state at any point), Resource allocation controls, Monitoring tools, Backup & replication features.
What is a Container?
Definition

A Container is a lightweight package that includes an application plus everything it needs to run (libraries, dependencies) — but it shares the host OS kernel instead of having its own full OS.

Simple Analogy: A VM is like renting an entire apartment (with its own kitchen, bathroom, etc.). A container is like renting just a room in a shared house — you share common areas (the OS kernel) but have your own private space (your app).
Why Use Containers?
  • Faster Startup — Containers start in seconds; VMs take minutes
  • Smaller Footprint — Container images are MBs; VMs are GBs
  • Portable — Run the same container on any system
  • Great for Microservices — Each service runs in its own isolated container
Container Technologies
Docker Engine Kubernetes (Orchestration)
Virtual Machine vs Container — Comparison
Feature Virtual Machine (VM) Container
Has its own Guest OS? Yes (full OS copy) No (shares host OS kernel)
Startup Time Slow (minutes) Fast (seconds)
Size Large (GBs) Small (MBs)
Isolation Level Hardware level Process level
Portability Moderate Highly portable
Best Use Case Running different OS environments Microservices, CI/CD, cloud apps
Containers in the Real World
  • Microservices (Netflix) — Netflix runs each service (login, recommendations, streaming) in separate containers. If one crashes, others keep working.
  • CI/CD Pipelines (eBay) — When code is pushed, containers automatically build, test, and deploy it. Fast rollbacks if something breaks.
  • Cloud-Native Apps (Uber) — Kubernetes orchestrates thousands of containers managing UberEATS, UberPOOL, and more across global cloud infrastructure.
📌 Topic 3 Quick Summary
  • Virtualization = running multiple virtual machines on one physical machine
  • A Hypervisor manages VMs; Type 1 runs on hardware directly (faster); Type 2 runs on an OS (easier)
  • Containers are lighter than VMs — they share the host OS kernel and start faster
  • VMs are isolated at hardware level; Containers at process level
  • Docker = container engine; Kubernetes = container orchestration (managing many containers)
  • Real-world users: Netflix (microservices), eBay (CI/CD), Uber (cloud-native)

Topic 04

Computer Architecture & Organization

CPU internals, the fetch-decode-execute cycle, memory hierarchy, and storage systems.
Key Distinction

Computer Architecture = the conceptual design of a computer system (think: blueprint). Computer Organization = how that design is actually implemented in hardware (think: construction). Architecture defines what it does; Organization defines how it does it.

4 Major Computer Components
CPU (Processor)
The brain. Executes all instructions and calculations.
Memory (RAM)
Temporary storage for currently running programs.
Storage
Permanent storage for data (HDD, SSD).
Input/Output Devices
Keyboard, mouse, monitor, printer — user interaction.
System Bus — The communication highway connecting all components. Carries three types of signals:
• Data Bus — transfers actual data  |  Address Bus — specifies memory location  |  Control Bus — sends commands
The CPU — Brain of the Computer

The CPU has 3 key internal parts:

ALU
Arithmetic Logic Unit Performs all math operations (add, subtract, multiply) and logical operations (AND, OR, NOT). Example: 5 + 3 = 8 is calculated by the ALU.
CU
Control Unit The manager of the CPU. Fetches instructions from memory, decodes them, and sends signals to other components to execute them.
REG
Registers Tiny, super-fast storage locations inside the CPU. Hold data temporarily during processing. Key registers: PC (Program Counter), IR (Instruction Register), MAR, MDR.
The Instruction Cycle — How the CPU Works
Also called

The Fetch–Decode–Execute Cycle. This is the fundamental loop the CPU repeats billions of times per second to run programs.

1
Fetch — The CPU retrieves (fetches) the next instruction from RAM
2
Decode — The Control Unit decodes the instruction to understand what operation to perform
3
Execute — The ALU or other components carry out the instruction
4
Store — The result is written back to memory or a register
Memory Hierarchy — Speed vs Cost vs Size

Computers use several types of memory. Faster = more expensive = smaller capacity.

Memory Type Speed Size Purpose
Registers Fastest Tiny (bytes) Immediate CPU operations
L1 Cache Very Fast Small (KBs) Most frequently used data
L2 / L3 Cache Fast Medium (MBs) Recently used data
RAM Moderate GBs Currently running programs
Secondary Storage Slow TBs Permanent file storage
Cache Memory is located physically close to the CPU and stores frequently-accessed data to reduce the time the CPU spends waiting for RAM. L1 is fastest; L3 is largest.
RAM is Volatile — it loses all data when you turn off the computer. Storage (HDD/SSD) is non-volatile — data stays even without power.
Storage Systems — HDD vs SSD
Feature HDD (Hard Disk Drive) SSD (Solid State Drive)
Technology Mechanical spinning disk Flash memory (no moving parts)
Speed Slower Much faster
Price Cheaper More expensive
Reliability Can fail (moving parts) More reliable
Best Use Large file storage (cheap) Operating system, fast apps

Other storage: USB Flash Drives, Optical Disks (CDs/DVDs)

Input / Output (I/O) Systems
Input Devices
  • Keyboard
  • Mouse
  • Scanner
  • Microphone
Output Devices
  • Monitor
  • Printer
  • Speakers
How I/O talks to the CPU: Through device controllers, drivers, and interrupts. An interrupt is a signal sent to the CPU saying "I'm done!" — so the CPU doesn't have to constantly check if I/O is finished.
📌 Topic 4 Quick Summary
  • A computer has 4 main parts: CPU, Memory, Storage, and I/O devices
  • CPU has 3 parts: ALU (math/logic), Control Unit (manages), Registers (fast temp storage)
  • CPU follows Fetch → Decode → Execute → Store cycle (billions of times per second)
  • Memory hierarchy from fastest to slowest: Registers → Cache → RAM → Storage
  • RAM is volatile (temporary); Storage is non-volatile (permanent)
  • SSD is faster and more reliable than HDD; HDD is cheaper and has more capacity per cost

Topic 05

Computing Infrastructure

Data centers, power and cooling systems, rack systems, and infrastructure management.
Key Definition

Computing Infrastructure refers to all the physical and virtual resources that support computing operations — servers, storage, networking equipment, power systems, and cooling. Think of it as the physical backbone of all IT systems.

Data Centers — The "Brain Hub"
Definition

A Data Center is a physical facility that houses computer systems and all related components. It's where all servers, storage, and networking equipment live. Without infrastructure, no apps or services can run.

What's Inside a Data Center?
Servers Storage Systems Networking Devices Power Supply Cooling Systems
Types of Data Centers
Type Who Owns It Description
Enterprise The company itself Private data center owned and operated by one organization
Colocation Shared facility Multiple companies share one building but have their own equipment
Cloud Cloud providers Managed by AWS, Microsoft Azure, Google Cloud. Rented virtually.
Power Requirements

Servers must run 24/7 without interruption. Power management is critical.

  • Power Consumption — Measured in Watts. Formula: Power (W) = Voltage × Current. Example: 220V × 5A = 1,100W
  • UPS (Uninterruptible Power Supply) — Backup battery that keeps servers running during a power outage (buys time for safe shutdown)
  • Generators — For extended outages; takes over when UPS runs out
  • Power Supply Units (PSU) — Internal power converters in each server
Total rack power = sum of all individual device wattages in the rack.
Cooling Requirements

Servers generate massive heat. Overheating destroys equipment. Target temperature: 18–27°C.

Air Cooling
Cool air is blown through the server room. Cheaper but less efficient for large setups.
Liquid Cooling
Liquid (water or coolant) runs through pipes to absorb heat. More efficient for dense setups.
Hot Aisle / Cold Aisle Design: Racks are arranged so cool air comes in from the cold aisle (front of rack) and hot air exits from the hot aisle (back of rack). Prevents hot and cool air from mixing, which improves efficiency.
Rack Systems
Definition

A rack is a standardized frame (usually 19-inch wide) used to mount and organize servers and equipment. Measured in Rack Units (U). 1U = one unit of height (~44mm).

  • 1U Server — Small, compact server (1 rack unit tall)
  • 4U Server — Larger server with more expansion space
  • Rack-mounted Servers — Standard servers mounted in a rack
  • Blade Servers — Ultra-thin modular servers that share power/cooling in one chassis
Benefits of racks: Organizes equipment neatly, improves airflow, and simplifies cable management and maintenance.
Infrastructure Management
Management Tasks
  • Monitoring systems
  • Regular maintenance
  • Security management
  • Backup and recovery
Management Tools
  • Nagios — Monitoring
  • Zabbix — Monitoring
  • Remote management systems
  • Asset tracking systems
Infrastructure Layout Design — Key Considerations
  • Space Planning — How much floor space is available? Where will equipment go?
  • Power Distribution — How is power routed to all equipment?
  • Cooling Placement — Where do cooling units go to maximize efficiency?
  • Cable Management — Organized cables prevent errors and improve airflow
Goal: Create an efficient, scalable, and maintainable infrastructure setup.
📌 Topic 5 Quick Summary
  • Computing infrastructure = all physical/virtual resources supporting IT operations
  • Data centers house servers, storage, networking, power, and cooling
  • 3 types: Enterprise (owned), Colocation (shared), Cloud (rented from AWS/Azure/Google)
  • Power: UPS (short outages) + Generators (extended outages); Power = Voltage × Current
  • Cooling: Keep servers between 18–27°C; use Hot/Cold aisle design for efficiency
  • Racks organize equipment; measured in Rack Units (U); 19-inch standard width

Topic 06

Platform Technology Optimization

Performance metrics, monitoring tools, optimization techniques, and troubleshooting.
Key Definition

Platform Performance refers to how efficiently a system (server, application, or infrastructure) responds to workloads. A high-performing system is fast, responsive, and stable. Poor performance leads to downtime, user frustration, and business losses.

Performance Metrics — What to Measure

Metrics are measurable values that tell you how healthy your system is.

Metric What it Measures Warning Sign
CPU Usage How much of the processor is being used >70% = possible bottleneck
Memory Usage How much RAM is consumed High = slow system or crashes
Disk I/O Speed of reading and writing to storage High activity = storage bottleneck
Network Latency Delay in network communication High = slow connections
Throughput Amount of data transferred per second Low = network congestion
Response Time How long a system takes to respond to a request High = poor user experience
Error Rate How often the system fails or returns errors High = reliability problem
Requests/sec How many requests the system handles each second Capacity limit indicator
Monitoring Tools

Tools that collect, store, visualize, and alert on system performance data.

Nagios
One of the oldest and most popular monitoring tools. Tracks servers, services, and networks.
Zabbix
Enterprise-level monitoring. Highly configurable; supports agent-based and agentless monitoring.
Prometheus
Modern, open-source metrics collection. Great for cloud-native and containerized environments.
Grafana
Visualization dashboard. Connects to Prometheus and others to display beautiful, real-time graphs.
How Monitoring Works: Collect data (metrics) → Store it → Visualize on dashboards → Trigger alerts when thresholds are exceeded.
Optimization Techniques
Goal

Optimization = making a system faster, more efficient, and able to handle more users by reducing inefficiencies.

1
Load Balancing Distributes incoming requests across multiple servers. No single server gets overwhelmed. Like having multiple checkout lanes at a store.
2
Caching Stores frequently-used data so it doesn't need to be recomputed. Example: A website caches your profile picture so it doesn't reload it every page.
3
Resource Scaling Adjust resources based on demand. Scale up = bigger machine. Scale out = more machines. Cloud makes this easy and automatic.
4
Code Optimization Improving the program's code to run more efficiently. Eliminate loops, reduce complexity, use better algorithms.
5
Database Tuning Improving how data is stored and retrieved. Add indexes, optimize queries, clean up unused data.
Bottlenecks — What Slows Systems Down
Definition

A Bottleneck is a single component that limits the entire system's performance. Like a traffic jam — even if all other roads are clear, one blocked lane slows everything.

  • CPU Bottleneck — Too many calculations; processor at 100%
  • Memory Bottleneck — Not enough RAM; system uses slow disk swap
  • Disk Bottleneck — Slow HDD can't read/write fast enough
  • Network Bottleneck — Congested network; slow data transfer
Troubleshooting Performance Issues — 5 Steps
1
Identify the Problem — What symptom is the user experiencing? Slow load? Crashes? Errors?
2
Analyze Metrics — Check CPU, memory, disk, network metrics to see which is abnormal
3
Locate the Bottleneck — Which specific component is the culprit?
4
Apply the Fix — Use appropriate optimization technique (scale, cache, tune DB, etc.)
5
Monitor Results — Verify the fix worked; continue monitoring to catch future issues
Example: A website is slow. CPU = normal, Memory = normal, Response Time = HIGH → Issue is network latency or backend processing delay, not hardware.
Best Practices for Platform Performance
  • Monitor Continuously — Don't wait for problems; detect them early
  • Use Alerts — Set thresholds so you're notified automatically when something is wrong
  • Optimize Regularly — Performance tuning is not a one-time task
  • Test Under Load — Simulate peak traffic before go-live to find bottlenecks in advance
📌 Topic 6 Quick Summary
  • Platform performance = how efficiently a system handles workloads (speed + stability)
  • Key metrics: CPU usage, RAM usage, Disk I/O, Network latency, Response time, Error rate
  • Monitoring tools: Nagios, Zabbix (collect + alert); Prometheus + Grafana (modern dashboards)
  • 5 optimization techniques: Load Balancing, Caching, Scaling, Code Optimization, DB Tuning
  • A bottleneck is one slow component that limits the whole system
  • Troubleshoot: Identify → Analyze Metrics → Find Bottleneck → Fix → Monitor

Topic 07

Platform Deployment & Integration

Deployment strategies, platform integration, cloud platforms, and cloud service models.
Key Definition

Platform Deployment is the process of installing, configuring, and making a system or application available for real-world use. It's the stage where development is finished and the system goes live. Example: Uploading your website to a web server so users can visit it.

Why Deployment Matters
  • Ensures System Availability — Users can't access the system without proper deployment
  • Supports Scalability — Good deployment plans for growth
  • Enables User Access — Makes the application reachable
  • Reduces Downtime — Structured deployment minimizes failures during rollout
Poor deployment = system failures, outages, and downtime.
Deployment Strategies — 4 Main Types
1
Recreate Deployment Shut down the old version completely, then launch the new version. Simple but causes downtime. Best for development environments.
2
Rolling Deployment Gradually update servers one by one (or group by group). Old and new versions run simultaneously for a short time. Less downtime than Recreate.
3
Blue-Green Deployment Two identical environments: Blue (current live) and Green (new version). Traffic switches from Blue to Green instantly. If Green fails, instantly switch back to Blue. Zero downtime. Easy rollback.
4
Canary Deployment Roll out the new version to a small percentage of users first (the "canary group"). If no issues, gradually increase to 100%. Like testing with a small audience before full release.
Strategy Downtime Rollback Risk
Recreate Yes Hard High
Rolling Minimal Moderate Medium
Blue-Green None Instant Low
Canary None Easy Very Low
Platform Integration
Definition

Platform Integration is the process of connecting different systems so they can work together and share data. Modern applications rarely work alone — they connect to databases, APIs, cloud services, and other systems.

Examples of Integration
  • Website + Database — Your blog posts are stored in a database that the website retrieves
  • Application + API — A weather app connects to a weather API to get current data
  • System + Cloud — Your company's app stores backups on AWS S3 cloud storage
Types of Integration
API Integration
Two systems talk to each other using APIs (Application Programming Interfaces). Most common type today.
Data Integration
Combining data from multiple sources into one unified view. Common in business analytics.
System Integration
Connecting entire systems/platforms together so they function as one cohesive unit. Example: CRM + ERP + HR systems all integrated.
Cloud Platforms
Definition

Cloud Platforms provide computing resources (servers, storage, databases, networking) over the Internet — so you don't need to own physical hardware. You rent what you need, when you need it.

☁️ Amazon Web Services (AWS)
Largest cloud provider. Offers 200+ services including EC2 (servers), S3 (storage), RDS (database).
☁️ Microsoft Azure
Microsoft's cloud. Tightly integrated with Windows, Office 365, and enterprise Microsoft products.
☁️ Google Cloud Platform (GCP)
Google's cloud. Known for data analytics (BigQuery), AI/ML tools, and YouTube infrastructure.
Cloud Service Models — IaaS, PaaS, SaaS

The three main ways cloud services are delivered, each with a different level of control:

Model Full Name What You Manage Real Example
IaaS Infrastructure as a Service You manage: OS, apps, data. Provider gives: servers, storage, networking. AWS EC2, Azure VMs, Google Compute Engine
PaaS Platform as a Service You manage: apps and data only. Provider handles: OS, runtime, servers. Google App Engine, Heroku, Azure App Service
SaaS Software as a Service You just use the application. Provider manages everything. Gmail, Google Docs, Microsoft 365, Dropbox
Simple Analogy — Pizza as a Service:
🏠 IaaS = You rent a kitchen and oven, buy ingredients, cook yourself.
🍕 PaaS = You get the kitchen ready-to-use, just bring your recipe and ingredients.
🚗 SaaS = Pizza is delivered to your door — you just eat it (no cooking needed).
📌 Topic 7 Quick Summary
  • Deployment = making a system live and available for users
  • 4 deployment strategies: Recreate (simple, has downtime), Rolling (gradual), Blue-Green (instant switch, zero downtime), Canary (test on small group first)
  • Blue-Green and Canary are best for production — minimal risk, easy rollback
  • Integration = connecting different systems to work together (via APIs, data sharing, etc.)
  • 3 cloud providers: AWS (largest), Azure (Microsoft), Google Cloud (analytics/AI)
  • IaaS = rent infrastructure | PaaS = rent platform | SaaS = use software directly
